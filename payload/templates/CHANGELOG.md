# Changelog

All notable changes to **{{PROJECT_NAME}}** are documented here.
The format follows [Keep a Changelog](https://keepachangelog.com/en/1.1.0/); versions follow
[Semantic Versioning](https://semver.org/).

Maintenance rule: every user-facing change lands an entry under `[Unreleased]` in the matching
category — **Added · Changed · Deprecated · Removed · Fixed · Security** — grouped, one heading
per category. Cutting a release renames `[Unreleased]` to `[x.y.z] - YYYY-MM-DD`, bumps the
version manifest in the same commit, and opens a fresh `[Unreleased]`. A pulled release keeps
its section, tagged `[YANKED]` after the date. Pre-first-release there
are no dated version sections: per Keep a Changelog, version headings correspond to real tagged
releases, so unreleased work stays under `[Unreleased]` until the first tag is cut.
